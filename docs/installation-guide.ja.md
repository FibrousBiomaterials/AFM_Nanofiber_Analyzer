# AFM Nanofiber Analyzer インストールガイド（共著者向け）

このドキュメントは、共著者がご自身の PC に **AFM Nanofiber Analyzer** を
インストールして起動できるようになることを目的とした、手順中心の解説です。
プログラミングの知識は前提としていません。表示されたとおりに上から順に
進めれば動作します。

- 主対象 OS は **Windows** です。macOS / Linux も最後に補足します。
- 一番かんたんなのは「**付属の `.bat` をダブルクリックする**」方法です
  （[2. かんたん手順（Windows / 推奨）](#2-かんたん手順windows--推奨)）。
- うまくいかないときは
  [6. うまくいかないとき（トラブルシューティング）](#6-うまくいかないときトラブルシューティング)
  を見てください。

技術的な背景や CLI・開発者向けの詳細は、リポジトリ直下の
[`README.md`](../README.md) / [`README.ja.md`](../README.ja.md) にまとまっています。
このガイドはそのうち「インストールして起動する」部分だけを、より丁寧に
書き下したものです。

---

## 0. 全体の流れ

インストールは大きく次の 3 ステップです。

1. **Python を入れる**（最初の一度だけ）
2. **本ソフトを入手する**（GitHub から ZIP ダウンロード、または `git clone`）
3. **セットアップ用ファイルをダブルクリックして実行する**（依存ライブラリの導入）

セットアップが終われば、2 回目以降は **起動用ファイルをダブルクリックするだけ**
で立ち上がります。

> インターネット接続が必要です（Python とライブラリのダウンロードのため）。
> 大学のプロキシ環境などで失敗する場合は、ネットワークの管理者にご相談ください。

## 1. 事前準備：Python のインストール

本ソフトは **Python 3.10 以上** が必要です（**3.11〜3.13 を推奨**。
動作確認はこの範囲で行っています）。

1. ブラウザで <https://www.python.org/downloads/> を開きます。
2. 「Download Python 3.x.x」ボタンからダウンロードします（推奨の
   3.11〜3.13 のいずれか。conda を使う場合は 3.11 が既定です）。
3. ダウンロードしたインストーラを実行します。
4. **【重要】最初の画面で「Add python.exe to PATH」（PATH に追加する）に
   必ずチェックを入れてから** 「Install Now」を押してください。

   ![Add to PATH のチェックを忘れない]

   ここにチェックを入れ忘れると、後の手順で「`py` が見つかりません」という
   エラーになります。その場合は Python を入れ直すか、PATH を手動で設定して
   ください。

5. インストールが終わったら、念のため確認します。スタートメニューで
   「PowerShell」または「コマンドプロンプト」を開き、次を入力して
   Enter を押します。

   ```powershell
   py -3 --version
   ```

   `Python 3.11.x` のようにバージョンが表示されれば成功です。

> すでに Anaconda / Miniconda を使っている方へ：
> Anaconda の `base` 環境からそのまま起動するのは**おすすめしません**
> （既存パッケージのバージョンが衝突することがあります）。
> conda を使いたい場合は
> [5. conda（Anaconda / Miniconda）を使う場合](#5-condaanaconda--miniconda-を使う場合)
> を参照してください。

## 2. 本ソフトの入手

リポジトリ：<https://github.com/q9-droid/AFM_Nanofiber_Analyzer>

入手方法は 2 通りあります。Git に慣れていない方は **方法 A（ZIP）** が簡単です。

### 方法 A：ZIP でダウンロード（Git 不要・初心者向け）

1. 上のリポジトリページを開きます。
2. 緑色の **「Code」** ボタン →「**Download ZIP**」をクリックします。
3. ダウンロードした ZIP を、分かりやすい場所（例：デスクトップやドキュメント）に
   **展開（解凍）** します。
4. 展開してできた `AFM_Nanofiber_Analyzer` フォルダを開きます。中に
   `Main.py` や `01_setup_venv.bat` などが並んでいれば OK です。

> 注意：フォルダのパスに日本語や空白が多く含まれていても通常は動きますが、
> 念のため浅い階層（例：`C:\Users\<ユーザー名>\AFM_Nanofiber_Analyzer`）に
> 置くと安心です。OneDrive 同期フォルダの中は、同期の影響で動作が不安定に
> なることがあるため避けるのが無難です。

### 方法 B：git clone（Git に慣れている方向け）

> ※ この方法には Git が必要です。未導入の場合は
> <https://git-scm.com/downloads> から先にインストールしてください。

```powershell
git clone https://github.com/q9-droid/AFM_Nanofiber_Analyzer.git
cd AFM_Nanofiber_Analyzer
```

## 3. かんたん手順（Windows / 推奨）

ここが本番です。プロジェクトフォルダ（`Main.py` がある場所）の中にある、
番号付きのバッチファイルを **エクスプローラーで順にダブルクリック** します。

### 3-1. セットアップ（最初の一度だけ）

`01_setup_venv.bat` を**ダブルクリック**します。

- 黒いウィンドウが開き、`[1/4] … [4/4] …` と進みます。
- このスクリプトは内部で次を自動的に行います。
  1. Python があるか確認する
  2. 本ソフト専用の隔離環境 `.venv` を作る
  3. `pip` を更新する
  4. 必要なライブラリ一式（NumPy, SciPy, scikit-image, matplotlib など）を入れる
- 最後に `Setup completed.` と表示されれば成功です。何かキーを押すか
  ウィンドウを閉じてください。

> `.venv` という専用環境を作るので、PC に元から入っている他の Python 環境を
> 汚しません。これが推奨される理由です。
>
> 途中で `Python launcher "py" was not found.` と出た場合は、手順 1 の
> 「Add python.exe to PATH」のチェックが漏れています。Python を入れ直して
> ください。

### 3-2. 起動（2 回目以降は毎回これだけ）

`02_run_from_venv.bat` を**ダブルクリック**します。

- ランチャー画面（プラグイン一覧のウィンドウ）が立ち上がります。
- ここから GUI01〜GUI04 の各ツールを開けます。

以後、ソフトを使うときは **`02_run_from_venv.bat` をダブルクリックするだけ**
です（`01` のセットアップを毎回する必要はありません）。

> 「ショートカットを作っておくと便利」：`02_run_from_venv.bat` を右クリック →
> 「ショートカットの作成」でデスクトップ等に置いておくと、毎回フォルダを
> 開かずに起動できます。

> SmartScreen の警告について：初回起動時に「WindowsによってPCが保護されました」
> と出ることがあります。これは署名されていないスクリプトに対する一般的な警告です。
> 内容を確認のうえ問題なければ「詳細情報」→「実行」で進めてください。

## 4. 各ツールの簡単な紹介

ランチャーから開けるツールの概要です（詳細は README を参照）。

| ツール | 役割 |
|---|---|
| **GUI01 Image Preprocessor** | 生の AFM データを読み込み、背景補正・二値化・細線化・キンク検出を行い、解析結果を `.b2z` バンドルとして保存します。まずここで前処理します。 |
| **GUI02 Plot Profiler** | 画像上で線分を引き、その断面（高さプロファイル）を取り出します。 |
| **GUI03 Fiber Height Histogram** | 複数の `.b2z` をグループ分けして、繊維の高さ分布を比較します。 |
| **GUI04 Fiber Tracker** | `.b2z` から個々の繊維を再構成し、繊維ごとの統計を CSV に出力します。 |

おおまかな流れは「**GUI01 で前処理 → 生成された `.b2z` を GUI02〜04 で解析**」です。
動作確認用のサンプルデータがリポジトリ内の `testdata_tunicateCNF/`（島津）と
`Bruker_testdata/`（Bruker）に同梱されています。

## 5. conda（Anaconda / Miniconda）を使う場合

すでに Anaconda / Miniconda を使っていて、そちらに合わせたい場合は、conda 用の
バッチファイルを使います。これは `base` 環境を変更せず、本ソフト専用の
`afm-analyzer` 相当の隔離環境（`.conda-env`）を作って、その中で動かします。

1. `11_setup_conda_env.bat` を**ダブルクリック**（最初の一度だけ・専用環境を作成）
2. `12_run_from_conda_env.bat` を**ダブルクリック**（以後の起動）

> `91_setup_anaconda.*` / `92_run_from_anaconda.*` は古い配布物との互換のために
> 残してあるだけです。既存の Anaconda 環境に直接インストールしてしまうため、
> 新規セットアップには**使わないでください**。

## 6. うまくいかないとき（トラブルシューティング）

| 症状 | 対処 |
|---|---|
| `01_setup_venv.bat` で「`py` was not found」と出る | 手順 1 の「Add python.exe to PATH」のチェック漏れです。Python を入れ直してください。 |
| 黒いウィンドウが一瞬で消える | ダブルクリック直後に閉じる場合、すでにセットアップ済みのことが多いです。`02_run_from_venv.bat` で起動できるか試してください。エラー内容を確認したいときは、PowerShell を開いてフォルダ内で `./01_setup_venv.bat` のように実行すると、ウィンドウが閉じずにメッセージが残ります。 |
| ライブラリのダウンロードに失敗する | ネットワーク（社内プロキシ等）の問題が多いです。別回線で試すか、ネットワーク管理者にご相談ください。 |
| 起動はするがウィンドウが出ない（GUI が表示されない） | tkinter が利用できない可能性があります。Windows の公式 Python では通常同梱されています。Linux の場合はガイド末尾の補足を参照してください。 |
| 一度入れた環境を作り直したい | プロジェクト内の `.venv`（conda なら `.conda-env`）フォルダを削除してから、もう一度 `01`（または `11`）を実行してください。 |

それでも解決しない場合は、**黒いウィンドウに表示されたエラーメッセージを
そのままコピーして** 開発担当（清遠）に共有してください。

## 7. 補足：macOS / Linux の場合

Windows の `.bat` に対応する `.sh` スクリプトが同梱されています。
ターミナルでプロジェクトフォルダに移動して実行します。

```bash
# 最初の一度だけ（実行権限を付与してからセットアップ）
chmod +x 01_setup_venv.sh 02_run_from_venv.sh
./01_setup_venv.sh

# 起動（2 回目以降はこれだけ）
./02_run_from_venv.sh
```

- macOS では Homebrew などで Python 3.10 以上を入れておいてください。
- Linux で GUI（tkinter）が見つからない場合、`01_setup_venv.sh` が
  `apt`/`dnf`/`yum`/`zypper`/`pacman` を検出して `python3-tk` 等の導入を
  試みます（`sudo` のパスワード入力を求められることがあります）。

## 8. アンインストール（元に戻す）

本ソフトはシステムに常駐インストールするものではありません。
専用環境はすべてプロジェクトフォルダ内に作られるため、**フォルダごと削除すれば
ほぼ完全に元に戻ります**（`.venv` や `.conda-env` も一緒に消えます）。
Python 自体を消したい場合は、Windows の「設定 → アプリ」からアンインストール
してください。
